# صيانة (Syanah)

منصة تجارية لخدمات الصيانة في السوق السعودي، تربط طالبي الخدمة بمزوّدين موثوقين عبر فئات متعددة (تكييف، سباكة، كهرباء، أجهزة منزلية، مركبات، صيانة منزل، وأكثر).

> **الحالة الحالية:** المرحلة 1 — البنية والأسس. لم يُكتب أي كود تطبيقي حتى الآن. مخرجات المرحلة 1 هي مخططات معمارية مُعدّة للمراجعة والاعتماد قبل بدء المرحلة 2.

## التسليم على مراحل

المشروع مُقسَّم إلى **12 مرحلة مُراجَعة**. انظر [`docs/PHASES.md`](docs/PHASES.md) للمُتابع الرئيسي للمراحل.

كل مرحلة يجب أن تكتمل وتُراجَع وتُختَبر (عند الانطباق) وتُعتمد قبل بدء التي تليها. الهدف هو تجنّب الأخطاء الشائعة عند بناء منصة كبيرة دفعةً واحدة: ميزات ناقصة، بنية غير مستقرة، متطلبات منسية، تكاملات معطوبة، أداء ضعيف، وكود يصعب صيانته.

## كيفية مراجعة المرحلة 1

1. اقرأ [`docs/architecture/00-overview.md`](docs/architecture/00-overview.md) أولاً.
2. تنقّل خلال المستندات من 01 إلى 12 بالترتيب.
3. استخدم [قائمة مراجعة المرحلة 1](docs/PHASE-1-REVIEW.md) لتسجيل الاعتمادات وأي طلبات تعديل.
4. بمجرد توقيع جميع البنود، يمكن بدء المرحلة 2 (نظام التصميم UI/UX).
