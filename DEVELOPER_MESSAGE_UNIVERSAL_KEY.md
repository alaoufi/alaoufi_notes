# رسالة جاهزة لمطوّر التطبيق (تحويل التفعيل إلى المفتاح العالميّ)

> انسخ ما تحت الخطّ وأرسله لمطوّر «حلالي» أو أيّ تطبيق آخر.

---

السلام عليكم. نريد توحيد نظام التفعيل في كل تطبيقاتنا على **مفتاح عالميّ واحد**
(Ed25519) بدل مفتاح خاصّ لكل تطبيق، فيُفعِّلها كلّها مولّدٌ واحد. **المطلوب تعديل
بسيط في كود التطبيق ثم إعادة بنائه ونشره.** المالك يحتاج المفتاح **العامّ** فقط
(أدناه)؛ المفتاح الخاصّ يبقى عند المالك في المولّد.

## الثوابت العالميّة (لا تتغيّر)
- المفتاح العامّ (Base64): `0JXPjbbPjczfYbYxl+jy1vOVcsEJT+CPbUIQgXNCStU=`
- بادئة الرسالة الموقَّعة: `UNIV1`
- ملح معرّف الجهاز: `alaoufi:`
- التعمية: Ed25519 — الترميز: Base32 `ABCDEFGHJKLMNPQRSTUVWXYZ23456789`

## إن كان التطبيق Flutter / Dart
في `lib/services/license_service.dart` اضبط الثوابت:
```dart
static const String _publicKeyB64 = '0JXPjbbPjczfYbYxl+jy1vOVcsEJT+CPbUIQgXNCStU=';
static const String _msgPrefix    = 'UNIV1';
// واشتقاق رقم الجهاز:
final digest = crypto.sha256.convert(utf8.encode('alaoufi:$raw'));
```
(أو استبدل الملفّ كاملًا بـ §4 من دليل «قاعدة التفعيل العالميّة».)

## إن كان التطبيق ويب / JavaScript
في كود التحقّق اضبط:
```js
const PUBKEY_B64 = '0JXPjbbPjczfYbYxl+jy1vOVcsEJT+CPbUIQgXNCStU=';
const PREFIX     = 'UNIV1';
// وفي اشتقاق رقم الجهاز استخدم الملح: 'alaoufi:'
```
(مرجع كامل: §8 من دليل «قاعدة التفعيل العالميّة».)

## ثم
1. أعِد بناء التطبيق ونشره (Flutter: `flutter build apk --release --split-per-abi`).
2. ثبّت النسخة الجديدة.

## ملاحظات
- صيغة الرسالة الموقَّعة: `UNIV1|رقم_الجهاز|المدّة` (المدّة بالأيام، 0 = دائم). يجب أن
  تطابق بادئة التطبيق المولّدَ حرفيًّا.
- **المستخدمون المفعَّلون حاليًّا لا يتأثّرون** — التفعيل المخزَّن لا يُعاد التحقّق منه؛
  التغيير يخصّ التفعيلات الجديدة فقط.
- رقم الجهاز يمرّ كما هو (يعرضه التطبيق → يُلصق في المولّد → يُوقّع → يتحقّق التطبيق)؛
  فيكفي ضبط المفتاح والبادئة لقبول الأكواد العالميّة.
