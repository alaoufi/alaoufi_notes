## 2. مراجعات معمارية

- [ ] **قابلية التوسع** — أهداف محددة للمستخدمين المتزامنين، ذروة الطلبات/الدقيقة، انتشار الدردشة. تحديد المسارات الساخنة. مراجعة استراتيجية التخزين المؤقت.
- [ ] **قاعدة البيانات** — مخطط مُطبَّع بشكل ملائم، فهارس مُوثَّقة، تغطية RLS كاملة، استراتيجية الترحيلات متّفق عليها.
- [ ] **الأمان** — مراجعة نموذج التهديد (STRIDE). معالجة البيانات الشخصية، التشفير في الراحة وفي النقل، خزينة الأسرار، سياسة تدوير المفاتيح. التوافق مع نظام حماية البيانات الشخصية السعودي (PDPL).
- [ ] **جدوى الـ Realtime** — اختبار Supabase Realtime لانتشار الرسائل المتوقع (محادثات × رسائل/ثانية). خطة بديلة عند بلوغ الحدود.
- [ ] **جاهزية المتاجر** — توثيق الأذونات، تخطيط privacy manifests، مراجعة سياسة المحتوى لـ iOS وAndroid وHuawei AppGallery.
- [ ] **التعدد اللغوي** — تدفّق الترجمة متّفق عليه، قواعد الـ fallback واضحة، تغطية RTL مكتملة.
- [ ] **نظام الثيمات** — كل عناصر الواجهة تقرأ من الرموز فقط (لا ألوان مضمَّنة). إضافة ثيمات مستقبلية لا تتطلب تعديل كود العناصر الأساسية.

## 3. قرارات مُقفَلة

القرارات التالية **مقفلة** عند الدخول للمرحلة 2. أي تغيير لاحق يحتاج إلى تعديل موثّق.

| المجال | القرار |
|---|---|
| إطار الويب | Next.js 15 (App Router) + TypeScript |
| التصميم | Tailwind CSS + متغيرات CSS لرموز الثيم |
| الموبايل | Flutter (iOS, Android, Huawei) — يُبنى بعد MVP للويب |
| قاعدة البيانات | Supabase Postgres (مُدارة) |
| Realtime | Supabase Realtime (نسخ Postgres + قنوات broadcast) |
| التخزين | Supabase Storage (حاويات خاصة + Signed URLs) |
| المصادقة | Supabase Auth + مُحوّلات تحقّق مخصّصة (نفاذ، SMS، WhatsApp) |
| الاستضافة | Vercel (الويب) + Supabase (البيانات) |
| الخرائط | Google Maps Platform |
| المدفوعات | Tap / Moyasar / HyperPay (يُحسم في المرحلة 9) |
| الإشعارات | FCM + APNs + HMS |
| المراقبة | Sentry + Vercel Analytics + Supabase logs |
| العملة | ريال سعودي (SAR) |
| المنطقة الزمنية | Asia/Riyadh (افتراضي على الخادم) |
| اللغات | AR (الافتراضية)، UR، EN، HI، BN |
| الثيمات | Soft Blue (الرسمي، الافتراضي)، Pink (الأنيق) |

## 4. أسئلة مفتوحة (يجب حسمها قبل التوقيع)

> أضف أي أسئلة غير محسومة هنا. أمثلة:

- [ ] مزوّد المدفوعات النهائي (Tap مقابل Moyasar مقابل HyperPay)؟
- [ ] مزوّد SMS OTP (تقنيات مقابل يونيفوني مقابل Twilio)؟
- [ ] منطقة استضافة Supabase (eu-central-1 مقابل me-south-1)؟
- [ ] اللغات المُفعّلة عند الإطلاق (AR + EN فقط، أم الخمس كلها)؟
- [ ] نموذج التسعير للمزوّدين (اشتراك، عمولة، هجين)؟
