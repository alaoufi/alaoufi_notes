# 00 — نظرة عامة على المنتج

## الرؤية

**صيانة (Syanah)** هي سوق إلكتروني يربط من يحتاج خدمات صيانة بمزوّدين موثوقين في جميع أنحاء المملكة العربية السعودية. مصمّم بمنهج Mobile-first، متعدّد اللغات، ويهدف للمنافسة على جودة تجربة المستخدم مع تطبيقات مثل Uber وكريم وAirbnb.

## النطاق (من MVP إلى المرحلة 12)

ضمن النطاق:

- سوق لخدمات الصيانة (تكييف، سباكة، كهرباء، أجهزة منزلية، صيانة منزل، مركبات، وفئات إضافية يحدّدها المشرف)
- تطبيق ويب (متجاوب، يضم بوابات للإدارة والعميل والمزوّد)
- تطبيقات موبايل أصلية (Flutter — iOS, Android, Huawei)
- دردشة لحظية لكل طلب مع وسائط وصوت وموقع
- تتبّع مباشر للمزوّد على Google Maps
- تقييمات ثنائية الاتجاه ومراجعات وحلّ نزاعات
- باقات اشتراك للمزوّدين وإعلانات منصة
- CMS إداري للترجمات والقوالب والإعدادات والإعلانات
- 5 لغات (AR, UR, EN, HI, BN) مع RTL/LTR

خارج النطاق في v1 (مؤجّل صراحةً):

- العمل خارج المملكة (المملكة فقط عند الإطلاق)
- مكالمات فيديو داخل التطبيق
- توصيات خدمة بالذكاء الاصطناعي
- بوابات White-label / B2B
- تطبيقات سطح مكتب macOS / Windows أصلية

## الأشخاص (Personas)

1. **طالب الخدمة** — العميل النهائي الذي يحجز الخدمات. التركيز: حجز سريع، تسعير واضح، إشارات ثقة.
2. **مزوّد الخدمة** — فني فردي أو شركة صغيرة. التركيز: استقبال الطلبات بسرعة، التنقّل، الأرباح.
3. **مشرف القسم** — موظف عمليات يدير الفئات والمزوّدين والنزاعات والمحتوى ضمن نطاقه المُسند.
4. **المشرف العام** — مالك المنصة بصلاحيات غير مقيّدة، تشمل العمليات المالية والأمنية.

## معايير النجاح (إطلاق المرحلة 12)

| البُعد | الهدف |
|---|---|
| زمن الحجز (طالب الخدمة) | < 60 ثانية من فتح التطبيق إلى إرسال الطلب |
| زمن قبول الطلب (المزوّد) | < 30 ثانية كوسيط |
| زمن وصول رسالة الدردشة | < 500 ms عند P95 |
| البدء البارد للتطبيق (موبايل) | < 2.5 ثانية على Android متوسط |
| LCP للويب | < 2.5 ثانية على 4G |
| CLS للويب | < 0.1 |
| الجاهزية (Uptime) | 99.9% شهرياً (باستثناء النوافذ المجدولة) |
| الجلسات بلا أعطال (Crash-free) | > 99.5% |

## القيود والثوابت غير القابلة للتفاوض

- **السوق السعودي أولاً.** جميع التدفّقات تدعم تجربة عربية أولاً، عملة SAR، منطقة Asia/Riyadh، عرض التاريخ الهجري حين يكون متوقّعاً ثقافياً.
- **التوافق مع PDPL.** معالجة البيانات الشخصية تلتزم بنظام حماية البيانات الشخصية السعودي.
- **لا توجد نصوص واجهة مكتوبة في الكود.** كل شيء قابل للترجمة.
- **لا توجد ألوان مُضمَّنة في الكود.** كل لون هو رمز تصميم (design token).
- **يجب أن تبقى البنية قابلة للتوسع.** ممنوع شعار "سنعيد كتابته لاحقاً".
- **تسليم مرحلي.** لا تبدأ مرحلة قبل مراجعة السابقة وتوقيعها.
