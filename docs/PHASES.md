# صيانة — المُتابع الرئيسي للمراحل

كل مرحلة لها بوابة. لا تُعتبر مكتملة إلا حين تُنجَز جميع بنودها وتُراجَع وتُختَبر وتُعتمد. "بوابة المرحلة" في أسفل كل مرحلة يجب أن تُوقَّع قبل بدء التالية.

دليل الحالة: `[ ]` لم تبدأ · `[~]` قيد التنفيذ · `[x]` منجزة · `[!]` متوقفة

## المرحلة 2 — نظام التصميم UI/UX  ·  **[x] مكتملة** — [مراجعة](PHASE-2-REVIEW.md)

**الهدف:** بناء نظام التصميم قبل الميزات — رموز التصميم، العناصر الأساسية، التخطيطات، ومحرّك الثيمات — كي تُعيد كل مرحلة لاحقة استخدام نفس اللبنات.

المخرجات:
- [ ] رموز تصميم (ألوان، تباعد، طباعة، أنصاف أقطار، ظلال، حركة) كمتغيرات CSS
- [ ] محرّك ثيمات: تبديل بين الأزرق الناعم والوردي في وقت التشغيل
- [ ] نظام طباعة (خطوط عربية + لاتينية، أحجام، ارتفاعات سطر)
- [ ] مقياس تباعد (شبكة 4px)
- [ ] عناصر أساسية قابلة لإعادة الاستخدام: Button, Input, Select, Card, Modal, Toast, Skeleton, EmptyState, Avatar, Badge, Tabs, Pagination, BottomSheet
- [ ] عناصر تخطيط: AppShell, BottomNav (جوال), TopNav (ويب), Sidebar (لوحة الإدارة)
- [ ] قواعد عكس RTL/LTR واستخدام خصائص CSS المنطقية (logical properties)
- [ ] فهرس Storybook لكل مكوّن
- [ ] تدقيق إمكانية الوصول (WCAG 2.2 AA): التباين، التركيز، لوحة المفاتيح، قارئات الشاشة

**بوابة المرحلة 2:**
- [ ] استجابة على الجوال / اللوحي / المكتب
- [ ] التحقق من RTL وLTR كليهما
- [ ] تبديل الثيم بدون أي تراجع بصري

- [ ] شاشات إدارة المستخدمين للمشرف العام فقط

**بوابة المرحلة 3:**
