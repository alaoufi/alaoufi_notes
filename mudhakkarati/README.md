# مذكراتي (Mudhakkarati / Alaoufi Notes)

تطبيق **مذكرات وملاحظات لأندرويد يعمل دون إنترنت بالكامل**، مبنيّ بـ **Flutter**،
بواجهة **عربية RTL** (مع دعم الإنجليزية)، يخزّن كل شيء داخل الجهاز في قاعدة
**SQLite مشفّرة (SQLCipher)** — بلا تسجيل دخول، وبلا خادم، وبلا تخزين سحابي إجباري.

> **لِمن هذا الملف؟** كُتب ليكون مرجعًا كاملًا ودقيقًا عن المشروع — تستطيع لصقه في أي
> مساعد ذكاء اصطناعي للحصول على أفكار جديدة، أو لمراجعة معماريّة، أو لتخطيط ميزات.
> آخِر قسم فيه («أفكار ومسارات تطوير») مُعدّ خصّيصًا لإشعال العصف الذهني.

- **اسم الحزمة:** `com.mudhakkarati.app`
- **الإصدار الحاليّ:** `1.7.0` (رقم البناء يُولَّد آليًّا = الدقائق منذ epoch)
- **الحدّ الأدنى:** Android 8.0 (API 26) — Flutter ‏`>=3.22`، Dart ‏`>=3.4`
- **التوزيع:** خارج Google Play (APK مباشر) + In-app update ذاتيّ، مع دعم App Bundle.

## 1) الفلسفة والمبادئ

| المبدأ | كيف يُطبَّق |
|---|---|
| **يعمل دون إنترنت 100%** | كل الميزات الأساسيّة لا تحتاج شبكة. الشبكة اختياريّة فقط للمزامنة السحابيّة والتحديث الذاتيّ. |
| **الخصوصيّة أولًا** | لا تتبُّع، لا تحليلات، لا حسابات. البيانات تبقى على الجهاز ما لم يُفعّل المستخدم المزامنة بنفسه. |
| **التشفير في كل طبقة** | قاعدة البيانات (SQLCipher) + كلمات المرور (مخزن مفاتيح) + النسخ الاحتياطيّة (AES‑256) + المزامنة (E2E). |
| **لا يتعطّل أبدًا** | الإقلاع محميّ بـ `runZonedGuarded`؛ أي خطوة تهيئة تفشل تُسجَّل ولا توقف التطبيق (`startupErrors`). |
| **عربيّ أصيل** | RTL كامل، واتجاه نصّ ذكيّ لكل سطر (bidi)، وتقويم هجريّ/ميلاديّ. |

## 2) الحزمة التقنيّة (Tech Stack)

- **الإطار:** Flutter (Material 3) + Dart.
- **إدارة الحالة:** `provider` (نمط `ChangeNotifier`).
- **قاعدة البيانات:** `sqflite_sqlcipher` (SQLite مشفّر) + `path`.
- **الإعدادات:** `shared_preferences`، والأسرار في `flutter_secure_storage`.
- **المحرّر الغنيّ:** `flutter_quill` (تخزين Delta JSON).
- **الإشعارات/المنبّه:** `flutter_local_notifications` + `timezone` + `flutter_timezone`.
- **الوسائط:** `image_picker`، `record` (تسجيل صوت)، `audioplayers`، `file_picker`،
  `open_filex`، `pdf` (توليد PDF)، `signature` (الرسم/الخط اليدويّ).
- **الإملاء الصوتيّ:** `speech_to_text` + قناة إملاء أصليّة (نظام أندرويد).
- **الأمان:** `local_auth` (بصمة/وجه)، `cryptography` و`encrypt` و`crypto`، `archive`.
- **المزامنة:** `googleapis` + `google_sign_in` (Google Drive)، وWebDAV عبر HTTP.
- **التقويم:** `table_calendar` + `hijri`.
- **الويدجت:** `home_widget`. **الألوان الديناميكيّة:** `dynamic_color` (Material You).
- **أدوات:** `intl`، `uuid`، `permission_handler`، `share_plus`، `url_launcher`،
  `package_info_plus`، `device_info_plus`، `android_id`، `flutter_staggered_grid_view`.

## 3) البنية المعماريّة

تقسيم طبقيّ واضح داخل `lib/`:

```
lib/
├── main.dart            # نقطة الإقلاع: تهيئة آمنة + حقن المزوّدات (MultiProvider)
├── app.dart             # MaterialApp + الثيم + التوطين + البوّابات (Gates)
├── core/                # ثوابت لا تعتمد على الميزات
│   ├── l10n/            # نظام توطين مخصّص (S) عربي/إنجليزي
│   ├── theme/           # الثيم، الألوان، التدرّجات، مخزن الألوان المخصّصة
│   ├── text/            # line_direction.dart — اتجاه كل سطر (bidi)
│   └── constants/       # أيقونات التصنيفات
├── data/                # طبقة البيانات
│   ├── models/          # نماذج غير قابلة للتغيير (Note, Category, Reminder, …)
│   ├── database/        # AppDatabase (SQLCipher) + مفتاح القاعدة (db_key)
│   └── repositories/    # وصول CRUD لكل كيان (Note/Category/Reminder/Med/Info/Log)
├── features/            # كل ميزة في مجلدها (شاشات + مزوّدات خاصة)
├── services/            # خدمات مستقلّة (Singletons) عبر التطبيق
└── widgets/             # عناصر واجهة مشتركة (بطاقة، حوارات، خلفية ورقيّة…)
```

### المزوّدات العامّة (Global Providers)
- `SettingsProvider` — الثيم، اللغة، الخط، طريقة العرض، تفضيلات شتّى.
- `NotesProvider` — تحميل/فلترة/فرز الملاحظات والتصنيفات والوسوم.
- `RemindersProvider` — التذكيرات + إعادة الجدولة الذاتيّة.

### تسلسل البوّابات عند الفتح
```
ActivationGate (ترخيص مربوط بالجهاز، يعمل دون إنترنت)
   └── AppLockGate (قفل التطبيق: رقم سرّي + بصمة)
          └── RootScreen → HomeScreen
```

### تسلسل التهيئة في `main.dart` (كلّها محميّة بـ `_safe`)
1. تهيئة تواريخ `intl` (عربي/إنجليزي).
2. `VaultService.ensureKey` — مفتاح تشفير كلمات المرور.
3. `NotificationService.init` + الأذونات + ربط فتح الملاحظة/شاشة المنبّه.
4. بناء المستودعات والمزوّدات.
5. `remindersProvider.ensureScheduled` — ضمان بقاء كل تذكير مجدولًا.
6. `MedDoseLogger.run` — تسجيل جرعات الأدوية الفائتة منذ آخر فتح.

## 4) نموذج البيانات

الكيانات الرئيسيّة (`lib/data/models/`):

- **Note** — العنوان، المحتوى (Delta JSON للنصّ الغنيّ)، النوع، التصنيف، الوسوم،
  الألوان/التدرّج، مثبّتة؟ مفضّلة؟ مقفلة؟ مؤرشفة؟ محذوفة؟، مرفقات (صورة/صوت/PDF/رسم)،
  تواريخ الإنشاء/التعديل/الحذف.
- **Category** — اسم، لون، أيقونة، ترتيب. (افتراضيّة: شخصي/عمل/مهم/مواعيد/أفكار، قابلة للتعديل.)
- **Reminder** — مرتبط بملاحظة أو مستقلّ، وقت، تكرار، أهميّة، حالة الجدولة.
- **ChecklistItem** — عنصر مهمّة (نصّ، `is_task`، `is_done`).
- **MedDose / med models** — جرعات الأدوية ومواعيدها وسجلّ أخذها/تفويتها.
- **PasswordEntry** — حقول منظّمة لكلمات المرور (الحقل الحسّاس مشفّر بالمخزن).
- **InfoEntry** — عناصر «قاعدة معلومات» عامّة (مرجع معرفيّ داخل التطبيق).
- **ReminderLogEntry** — سجلّ أحداث التذكيرات (للموثوقيّة والتشخيص).

**أنواع الملاحظات** (`NoteType`): `text`، `checklist`، `image`، `audio`، `pdf`،
`drawing`، `password`.

**القاعدة:** `AppDatabase` تفتح SQLite عبر SQLCipher بمفتاح من `db_key` (مشتقّ/مخزَّن
بأمان)؛ كل الجداول داخل الملف المشفّر. الترقية تُدار بنسخ ترحيل (migrations).

## 5) كتالوج الميزات الكامل

### 🏠 الرئيسية والتصفّح
- بطاقات ملوّنة بشبكة **Masonry** (أو قائمة)، الملاحظات المثبّتة أولًا.
- شريط تصنيفات أفقيّ مع عدّاد لكل تصنيف، زرّ إضافة عائم (FAB)، زرّ «للأعلى» يظهر عند التمرير.
- **بحث فوريّ** (عنوان + محتوى) و**فلترة متقدّمة** (نوع، صورة/صوت/PDF، مثبّت/مقفل، تاريخ).

### ✍️ المحرّر الغنيّ
- تنسيق كامل: غامق/مائل/تسطير/شطب، لون نصّ، **تظليل (قلم باستيل ناعم)**، خطّ وحجم،
  محاذاة، قوائم نقطيّة/رقميّة، اقتباس، تباعد أسطر.
- **اتجاه ذكيّ لكل سطر** حسب **اللغة الغالبة** (يمنع تلخبط الأسطر المختلطة عربي/إنجليزي).
- **إملاء صوتيّ** (تحويل الكلام إلى نصّ على الجهاز)، إدراج مرفقات، **مشاركة كصورة**.
- حفظ تلقائيّ مؤجَّل أثناء الكتابة (debounce)، وتخزين بصيغة Delta دون سمة الاتجاه (تُحسب عند العرض).

### 🗂️ التنظيم
- تصنيفات قابلة للإدارة (لون/أيقونة/ترتيب)، **وسوم (Tags)** وفلترة بها، **روابط بين الملاحظات**،
  **قوالب جاهزة** للملاحظات.

### ⏰ التذكيرات والمنبّهات
- تذكير لأي ملاحظة أو تذكير **مستقلّ**، تكرار: مرّة/يوميّ/أسبوعيّ/شهريّ/سنويّ.
- **مستويات أهميّة**: `low` (صامت)، `medium` (صوت)، `high` (صوت+اهتزاز)،
  `critical` (**منبّه حقيقيّ**: شاشة كاملة + إصرار حتى التفاعل).
- مركز إشعارات، إعدادات افتراضيّة للتذكير، **شاشة اختبار موثوقيّة**، إعادة جدولة ذاتيّة
  عند الإقلاع (لا تضيع التذكيرات)، مكتبة نغمات + معاينة + منتقي نغمة النظام + ضبط مستوى صوت المنبّه.

### 💊 الأدوية
- جدول أدوية بمواعيد جرعات، **تسجيل تلقائيّ** للجرعات المأخوذة/الفائتة في سجلّ.

### 📅 التقويم والرؤى
- عرض الملاحظات/التذكيرات حسب التاريخ بتقويم **ميلاديّ + هجريّ**.
- **ملخّص أسبوعيّ** (Insights) للنشاط.

### 🎨 الوسائط
- صور، تسجيل صوتيّ + تشغيل، إرفاق PDF وفتحه، **رسم/كتابة يدويّة** (ألوان/أحجام أقلام/تراجع/مسح).

### 🔒 الأمان والخصوصيّة
- قفل التطبيق (**رقم سرّي + بصمة/وجه**)، قفل ملاحظات أو تصنيفات بعينها، **ملاحظات سرّية**.
- **شاشة آمنة** (`FLAG_SECURE`) تمنع لقطات الشاشة في الأماكن الحسّاسة.
- **مخزن كلمات مرور** مشفّر داخل ملاحظات نوع `password`.

### ☁️ المزامنة والنسخ الاحتياطيّ
- **نسخ احتياطيّ محليّ مشفّر** (AES‑256، مؤرشَف) + **نسخة يوميّة تلقائيّة**.
- **مزامنة سحابيّة E2E** عبر **WebDAV** أو **Google Drive** (الملف مشفّر بعبارة سرّ المستخدم).
- **خيارات تردّد المزامنة**: كلّ فتح / عند الإغلاق / مرّة باليوم، مع **وضع صامت في الخلفية**.
- **استيراد** ملاحظات Easy Notes من نسخها الاحتياطيّة `.backup` مع الحفاظ على التنسيق.

### 🧩 تجربة وإعدادات
- ويدجت الشاشة الرئيسيّة (آخر/أهمّ ملاحظة)، سلة محذوفات + أرشيف + تنظيف.
- ثيم فاتح/داكن، **ألوان النظام الديناميكيّة (Material You)**، لون بذرة + ألوان مخصّصة،
  عائلة الخطّ وحجمه، طريقة العرض، اللغة، **دليل مساعدة** (19 موضوعًا).

### 🔁 التحديث الذاتيّ
- مقارنة رقم البناء المحليّ بـ `version.json` المنشور، تنزيل APK وتشغيل **مثبّت النظام
  الأصليّ** (قناة `installer` + FileProvider + إذن `REQUEST_INSTALL_PACKAGES`).
  المصادر بالترتيب: `raw.githubusercontent` ← `jsDelivr`، والتنزيل من GitHub Releases.

## 6) نموذج الترخيص (Activation)

تفعيل **مربوط بالجهاز يعمل دون إنترنت**:
- لكل جهاز **معرّف ثابت (Device ID)** مشتقّ من عتاده ويُعرض للمستخدم.
- **رمز التفعيل = توقيع Ed25519** (تشفير غير متماثل) على «معرّف الجهاز + المدّة»،
  يولّده المالك بمفتاحه الخاصّ عبر **تطبيق مولّد مستقلّ**؛ والتطبيق يتحقّق بالمفتاح العامّ.
- حالات: غير مضبوط (وضع تطوير مفتوح) / لم يُفعّل / مفعّل (بمدّة أو دائم) / منتهٍ.
- يُعاد الفحص عند عودة التطبيق للواجهة كي يُطبَّق الانتهاء فورًا.

## 7) خطّ البناء والنشر (CI/CD)

GitHub Actions: `.github/workflows/build-apk.yml` (يُشغَّل على الدفع):
1. **`flutter test`** — تفشل الخطوة كلّها إن سقط أي اختبار (بوّابة جودة قبل النشر).
2. **بناء**: `flutter build apk --release --split-per-abi --obfuscate` + `appbundle`.
   - `versionCode = (epoch ثوانٍ ÷ 60)` — رقم بناء تصاعديّ آليّ (`--build-number`).
3. **نشر** على فرع `apk-dist`: `version.json` (`{version, build, url}`) + APK، وعلى
   **GitHub Releases** (وسم `latest`).
4. **سجلّ تشخيصيّ**: يُدفع `build.log` إلى فرع `build-log` دائمًا (حتى عند الفشل).

> ⚠️ **درس مستفاد:** فشل الاختبارات يوقف النشر بصمت ⇒ يبقى `version.json` قديمًا ⇒ لا يظهر
> تحديث داخل التطبيق. **القاعدة:** شغّل `flutter test` محليًّا قبل كل دفع.

### بناء محليّ

cd mudhakkarati
flutter pub get
flutter test                                   # بوّابة الجودة
flutter run                                     # تجربة على جهاز
flutter build apk --release --split-per-abi     # APK لكل معماريّة

## 10) أفكار ومسارات تطوير (للعصف الذهني مع AI)

> هذا القسم مقصود ليكون **نقطة انطلاق**؛ اعرضه على المساعد واطلب توسيعه أو تحدّيه.

### حدود حاليّة جديرة بالمعالجة
- **المزامنة** تجري على الـ isolate الرئيسيّ (تصدير/تشفير) ⇒ قد تُحدث تقطّعًا بسيطًا مع آلاف
  الملاحظات؛ مرشّحة للنقل إلى `compute`/isolate.
- **«المزامنة عند الإغلاق»** أفضل جهد على أندرويد (قد لا تكتمل عند تجميد النظام للتطبيق).
- لا يوجد **حلّ تعارضات** متقدّم في المزامنة (آخر كتابة تغلب) — يمكن تحسينه.
- البحث `LIKE` نصّيّ بسيط (لا فهرسة FTS ولا مطابقة عربيّة بلا تشكيل/همزات).

### أفكار ميزات محتملة
- **بحث دلاليّ/كامل النصّ** (SQLite FTS5) مع تطبيع عربيّ (تجاهل التشكيل وتوحيد الألف/الهمزة).
- **مساعد ذكاء اصطناعيّ على الجهاز**: تلخيص ملاحظة، اقتراح عنوان/وسوم، تحويل ملاحظة إلى مهامّ.
- **علاقات الملاحظات** كرسم بيانيّ (Backlinks/Graph) اعتمادًا على `note_links`.
- **مزامنة فوريّة لكل تغيير** (delta sync) بدل المزامنة الكاملة، مع حلّ تعارضات بالطوابع الزمنيّة.
- **تشارُك مشفّر** لملاحظة واحدة (رابط/QR مع عبارة سرّ).
- **تذكيرات ذكيّة** قائمة على الموقع (geofence) أو على سياق (نصّ يحتوي تاريخًا ⇒ اقتراح تذكير).
- **لوحة معلومات الأدوية**: التزام الجرعات، رسوم بيانيّة، تنبيه نفاد المخزون.
- **قوالب أغنى** + حقول مخصّصة، و**تصدير/استيراد Markdown**.
- **ويدجتات متعدّدة** (مهامّ اليوم، التذكير القادم، عدّاد)، و**Quick Settings tile**.
- **وضع تركيز/كتابة** و**اختصارات لوحة مفاتيح** للأجهزة اللوحيّة.
- **تحسين إمكانيّة الوصول** (TalkBack، تباين، أحجام لمس).

### أسئلة جيّدة لطرحها على AI
- ما أكثر ميزة سترفع الاحتفاظ بالمستخدم دون كسر مبدأ «دون إنترنت/خصوصيّة أولًا»؟
- كيف أضيف ذكاءً اصطناعيًّا **على الجهاز** (بلا خادم) ضمن قيود أندرويد؟
- ما أبسط طريق موثوق لمزامنة فوريّة مع حلّ تعارضات على WebDAV/Drive؟
- كيف أحسّن البحث العربيّ (تشكيل/همزات/جذور) بأقلّ تعقيد؟
